## Known test-infra issues

### Deadlock retry now catches the adapter-wrapped form (¬ßB ‚Äî addresses a prior carryover)

The 2026-05-14 cycle saw transient `TransactionWriteConflict` failures on `resetUser`
against the remote DB. Root cause: `@prisma/adapter-pg` wraps Postgres `40P01` as a
`DriverAdapterError` whose typed `code` is undefined, so the `P2034 / 40P01` matcher
never engaged. ¬ßB widens the matcher (both `test-control.service.ts` and the shared
`common/deadlock-retry.ts`) to also catch the `TransactionWriteConflict` message
substring ‚Äî conservative widening, does not blanket-retry unrelated errors. This should
reduce the transient-conflict flake class on remote-DB local runs.

### Shared seed DB pollution + Aisha data wiping

Unchanged from 2026-05-14. Local runs against the shared remote DB hit seed-state
pollution and archetype wiping; CI uses fresh DB-per-shard. The `Carol Miller` patient-list
assertion (`spec 10:101`) fails locally on the swapped DB because the Cluster 6 persona
expansion (Carol/Mike/Kate/‚Ä¶) is not seeded there ‚Äî **pre-existing seed gap, unrelated to
this PR**.

### Backend `tsc --noEmit` non-zero exit is pre-existing spec-typing noise

`cd backend && npx tsc --noEmit` exits non-zero, but **every error is in `*.spec.ts`
files** (jest mock `Argument of type 'X' is not assignable to parameter of type 'never'`)
or the unrelated `test/llm-judge/voice-chat.e2e-spec.ts` ‚Äî all present on `dev` before this
PR. **This PR's changed source files produce zero tsc errors** (verified by filtering).
