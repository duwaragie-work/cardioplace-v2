## Accepted WCAG debt (NOT regressions)

Per the vibrant red/amber token decisions (commits `43e4aa2`, `6f6cdf3`, `70f2ff4`), some axe-core violations are **explicitly accepted** as known debt. These are excluded from spec 16's `.toEqual([])` assertion via two mechanisms:

**1. Component-level `data-axe-debt` attributes** — for explicit point-of-use tagging:
- `data-axe-debt="avatar-orange-small-text"` — patient initials circles
- `data-axe-debt="severity-pill-orange-small"` — tier badges <14px
- `data-axe-debt="status-dot-orange"` — status indicator dots
- (etc. — search for `data-axe-debt` in `frontend/src/` and `admin/src/` for the full list)

**2. CSS attribute selectors** — for chip-on-tinted-bg pattern (long tail):
```ts
'[style*="var(--brand-warning-amber-light)"][style*="var(--brand-warning-amber-text)"]'
'[style*="var(--brand-alert-red-light)"][style*="var(--brand-alert-red-text)"]'
```

**Why:** orange-500 (#F97316) on amber-100 (#FEF3C7) at <14px = 2.51:1 — fails AA Normal. Mitigation plan is **font-size bumps to ≥14px bold** to satisfy AA Large, NOT color rollback. Tracked as Cluster 7+ polish (item not on Niva's blocker list — frontend / Lakshitha).

If a NEW a11y issue appears outside the accepted-debt patterns, spec 16 WILL catch it — exclusion is scoped, not blanket.

## Known test-infra issues (carryover from prior cycle)

### Shared seed DB pollution

Most failures observed during local Playwright runs against the remote `db.prisma.io` are **pre-existing seed-state pollution** — tests mutate state, leave it behind, subsequent runs find a different state than setup assumes. Symptoms: tests that pass in isolation fail when run in sequence. Root cause: no per-test DB isolation.

**Workarounds in use:**
- Per-run-unique values (timestamps, derived DOBs) for write-side tests
- CI uses fresh DB-per-shard (no pollution); local runs against shared dev DB do hit this

**Long-term fix (Cluster 8+ infrastructure):**
- Ephemeral schemas per test invocation
- Or test-container DB seeded fresh on each `playwright test` invocation

### Aisha manual-test data wiping

Playwright suite cleanup wipes patient state for seeded archetypes (Aisha, James, Paul, Nora, Robert, etc.) when tests run. Manual testing data on the same DB will be lost. **Workaround:** for manual smoke during dev, create a fresh test patient (e.g., "ManualTest1") that no spec references.
